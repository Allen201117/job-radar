## 2. 缺口诊断（为什么剩下的难 = 中国500强的主体恰恰难）
中国500强主体 = **重工业 / 能源 / 电信 / 建筑 / 资源 / 银行央国企 + 传统制造**，它们的招聘系统分四类：

| 类 | 系统 | 现状 | 代表 |
|---|---|---|---|
| **A** | hotjob **wt/wintalent** 老形态 `{brand}.hotjob.cn/wt/{BRAND}/` | adapter **只支持 wecruit 新形态，wt 不支持** | 中国中车 crrc、银河证券 Chinastock，一批 SOE |
| **B** | 北森 zhiye.com | adapter 支持，需逐家探 slug + 补 /campus | 部分能源/建筑/制造国企 |
| **C** | 大易 dayee.com | **无 adapter** | 部分制造（隆基/比亚迪曾疑似，需 live 复核） |
| **D** | 完全自建 SPA/门户 | 每家 bespoke，**详情页深链可行性须逐家验证** | 中石化/国网/中国建筑/四大行/美的(已验不可行)/格力/立讯/伊利 |

**红线**：拿不到稳定逐岗 `jd_url` 的源不入 active（美的 `recruit.midea.com` 已验证：有干净岗位 JSON 但详情页 SPA 无公开深链→不做，只能记 partial）。

## 3. 攻坚优先级（先解锁一类，再铺量；按 ROI 排）

> **2026-06 更新（用户定原则，最高优先）**：
> 1. **P0(wt 老形态) 与 P1(北森 SOE) 已攻克并接线**——`crawler/adapters/wt.py`（直连 position/list JSON + postId-only jd_url）、beisen/feishu/moka 三个 adapter 的列表分页截断 bug 已全修（放大全部存量源）。
> 2. **私企 500强 > 国企央企 500强**：国企央企**只顺手收「已落在已攻克平台（wecruit/wt/北森/moka/飞书）」的**；**完全自建门户 / 国聘 iguopin 聚合 的硬骨头央企先不管**（深链不稳、易撞红线、岗位不贴产品用户）。**精力优先投私企 500强**。
> 3. **当前重排**：🟢T1 私企+SOE-易（已解决平台，零代码 probe+emit）→ 🟡T2 私企需中等 adapter（大易 dayee 等）→ 🔴T3 自建硬骨头（多撞红线，缓做，能深链才做）。
> 下面 P0–P3 为历史攻坚记录（P0/P1 已完成）。

### 🥇 P0（最高 ROI，先做）— 扩 hotjob 适配器支持 wt/wintalent 形态
- **价值**：一次解锁中车/银河 + 一大批用 wt 的央企 SOE（央企爱用 wt）。
- **做法**：
  1. spike 逆向 `https://crrc.hotjob.cn/`（中车）与 `https://www.hotjob.cn/wt/Chinastock/...`（银河）的**岗位列表接口**（不是 wecruit 的 `/wecruit/positionInfo/listPosition/`，wt 走另一套，常见 `/wt/{BRAND}/...` 下的 search/position API）。用 playwright 拦截或直接抓 XHR。
  2. 找到 list JSON 接口特征 + 详情页 URL 模板（postType/recruitType 分社招/校招/实习）。
  3. 在 `crawler/adapters/hotjob.py` 加 wt 分支（或新 `hotjob_wt` adapter，复用 playwright_base）。
  4. 三渠道：wt 的 recruitType 参数分社招/校招/实习，一并支持。
- **验收**：crrc live 返回真岗 + jd_url 可点 + 岗位内容 title-verify 是中车。

### 🥈 P1 — 北森 SOE 扩量 + 补三渠道
- **方法**：WebSearch `{公司} zhiye.com`、`zhiye.com 社会招聘 央企/能源/建筑/钢铁`。**zhiye 通配恒返回 200**，靠门户 `<title>` ≠ `Not Found` 判真伪（防张冠李戴）。
- **三渠道**：每家入 `/social` + `/campus`（+ 实习若有独立板块）；新详情路由落 `crawler/beisen_routes.json`。
- **注意**：新版拦 `GetJobAdPageList`；C 型老 SSR 走 `_fetch_ssr`（HTML 锚点）。

### 🥉 P2 — 大易 dayee.com 通用 adapter
- 先 live 复核真实客户招聘页（隆基/比亚迪等，逐家验证不是品牌页），逆向 dayee 公开岗位 JSON，建通用 adapter（host 从 source_url 动态解析，参考 china_ats.py 三个通用 adapter 的写法）。

### P3 — 自建巨头 bespoke（最硬，按 job 量排，能不做先不做）
- **每家先验证「详情页是否公开可深链」**，深链不了=红线外=不做（记 partial）。
- 方法：playwright 拦截官方岗位接口（参考 `crawler/adapters/bytedance.py` / `feishu.py` 范式）。
- 候选按 job 量：中石化/国网/中国建筑/工行 等；**美的=已验证不可行，别重做**。

## 4. 通用方法与铁律（每个 adapter 都遵守）
1. **探活入库**：`cd crawler && python3 probe.py --all --candidates X.json --emit 0NN`。候选 JSON `{company, adapter, url, industry?}`；只把**真返回岗位**的 emit 进 `supabase/migrations/0NN_seed_probed_sources.sql`，push 自动迁移。china-gate：本土需 valid>0。
2. **张冠李戴铁律**：每个新源/suiteKey/slug **必 title-verify**（门户 title 或岗位内容含该公司名）。WebSearch 常混入**通用「社会招聘」链 = 毒**（同一个 SU…/zhiye slug 在多公司搜索里都出现）。
3. **jd_url 质量门**：`company/title/jd_url` 非空 + 官方域 + 唯一 id；拿不到稳定逐岗链 = 只记 `partial_success` 不入 active。
4. **三渠道**：社招/校招/实习是**独立入口**，逐家三个都入（hotjob 换 page、moka 换 orgId、北森换板块、wt 换 recruitType）。
5. **落注册表防丢**：hotjob suiteKey → `hotjob_suitekeys.json`；北森路由 → `beisen_routes.json`。
6. **入库即自动**：push 触发 `migrate.yml` + 下次 `daily-crawl`（手动触发：`gh workflow run 287499240`）。前端 `lib/china-keyword-expansion.js` 的 `recruitmentCategory` 已按 jd_url 的 campus/intern 标记自动归三桶。
