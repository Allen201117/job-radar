# 求职雷达产品与工程加固设计

**日期：** 2026-07-10  
**基线：** `origin/main`  
**目标：** 先修复会破坏用户信任或造成生产风险的问题，再处理性能与可访问性；本轮不新增业务模块。

## 1. 范围

本轮只实施四组能在仓库内验证的改动：

1. 两个岗位探活接口必须验证真实 Supabase 用户，服务端埋点只接受验证后的用户 ID。
2. 今日机会按“公司 + 标题 + 地点”做语义去重；在候选充足时优先保证公司多样性，在候选不足时允许回填，不能为了配额制造空列表。岗位卡明确区分“官网发布”和“最近确认仍开放”，公司岗位数使用有效在招口径。
3. 岗位库统计改为 CDN 可缓存并降低自动轮询频率；修复标签输入框、移动菜单遮罩和已投递空状态的关键体验问题。
4. 升级存在已知高危漏洞的 Next.js 依赖；补齐数据库 TLS、备份和恢复演练的运维核验门，但不在没有生产证书和云平台权限时声称基础设施已经完成。

## 2. 非目标

- 不重做页面视觉风格。
- 不新增职业路径、洞察、海外岗位或商业化功能。
- 不执行生产探活攻击、生产写库测试或数据库恢复操作。
- 不引入 Redis、队列或新的监控供应商。
- 不把关键提醒区按公司限流；收藏岗位关闭、快截止等关键提醒必须完整保留。

## 3. 关键设计

### 3.1 探活鉴权

批量与单岗探活路由首行调用 `requireUser()`。验证失败立即返回统一 401，且不得创建 service-role client、解析请求体、访问岗位库或发起外部请求。单岗探活使用 `auth.user.id` 写 `job_liveness_at_click`，移除 `getRequestUser()`，因为 `/api/*` 不经过页面中间件。

### 3.2 语义去重与软性公司配额

语义键由标准化后的 `company/title/location` 组成；字段缺失时回退到岗位 ID，避免把信息不足的岗位误合并。相同语义键保留排序更靠前的机会。

公司多样性只作用于 `main` 和 `explore`：

- 第一轮每家公司最多占区块目标数量的 30%，且至少可取 2 条；
- 第二轮用溢出候选回填空位，因此候选只有一家时仍能填满列表；
- `critical` 与 `waiting` 不做公司配额；
- 去重发生在分区前，确保同一语义岗位不会跨区重复。

### 3.3 时间与计数口径

岗位元数据中的 `posted_at` 显示名改为“官网发布”；现有 `last_seen_at` 新鲜度文案继续表达“最近确认仍在招”。`active_job_counts_by_company()` 增加与 `count_valid_active_jobs()` 相同的正文不少于 60 字条件。

### 3.4 统计缓存与基础可访问性

`/api/jobs/stats` 返回 `public, s-maxage=60, stale-while-revalidate=300`，客户端每 60 秒自动刷新；用户仍可手动刷新。`TagInput` 增加必填的 `ariaLabel`，所有调用点给出业务名称。移动菜单的视觉遮罩从可聚焦按钮改为 `aria-hidden` 的非语义层。已投递空状态使用真实按钮名“标记投递”，并提供返回“今日机会”的唯一主行动。

### 3.5 依赖与运维

依赖升级以 `npm audit --omit=dev` 的高危项归零为安全门，并完整运行 Node、Python、lint、build 和迁移检查。数据库运维文档必须明确：可信 CA、禁止长期使用 `rejectUnauthorized:false`、每日备份、PITR、RPO/RTO、季度恢复演练及证据位置。

## 4. 失败与降级

- 身份验证服务异常时，探活返回认证错误，不得 fail-open 写共享数据。
- 统计接口失败时沿用前端 stale 状态和 SSR 初始值。
- 多样性候选不足时回填，不降低可用机会数量。
- Next.js 升级若导致构建或核心测试失败，保留在修复分支，不合并、不部署。
- 未配置可信 CA 前，仅记录生产阻塞项，不静默切换 TLS 行为导致线上断库。

## 5. 验收

- 伪造 Cookie 不能通过两个探活路由；未认证路径无任何外部或数据库副作用。
- 单岗探活事件的 user ID 来自 `requireUser()`。
- 相同公司、标题、地点的不同 ID 岗位只出现一次。
- 候选公司充足时主区块单公司占比不超过 30%（最少配额 2）；只有单一公司时不产生空列表。
- 岗位卡同时具备“官网发布”和“确认仍在招”的清晰语义。
- 公司岗位数只统计有效在招岗位。
- 统计接口具备 CDN 缓存，自动轮询为 60 秒。
- 关键输入有可访问名称；移动遮罩不可聚焦；已投递空状态有清晰 CTA。
- 生产依赖无 high/critical audit 项；全量验证通过。
