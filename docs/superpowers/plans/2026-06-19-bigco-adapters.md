# BigCo Adapters Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add production crawler support for Meituan, Kuaishou, bilibili, Pinduoduo, vivo, and BYD with verified per-job URLs, while documenting SF Express as currently infeasible.

**Architecture:** Use one focused adapter per company. Meituan, bilibili, Pinduoduo, and vivo use anonymous public HTTP APIs; Kuaishou uses Playwright so the public page can generate its request signature; BYD uses Playwright to click first-page rows and capture the site-generated encrypted detail URLs. All adapters emit `RawJob` objects and rely on the shared normalizer quality gate.

**Tech Stack:** Python 3.9, httpx, Playwright sync API, unittest, Supabase seed migrations, TypeScript adapter metadata.
