## 0. 执行摘要

产品 = **一份能投的官网机会清单**。三层：

1. **地基（护城河）**：官方源 + 持续核验 + 极少死岗（可测，不吹绝对）；
2. **产品形态**：适合你、刚核验还在招的机会清单（现在主要靠"对口 + 保鲜"填）；
3. **辅助信号（降级、先当标签不投入）**：我们新发现 / 快截止 / 收藏岗关闭 / 校招通道开放 / 公司招聘动量。

不要模式。雷达靠**身份（简历来）× 强度（自调）**运转。

---

## 1. 不做什么（本轮）

自动投递、HR 直聊、绕验证码、第三方平台覆盖对比、"岗位内容变更"提示、"重新开放"提示、大规模铺源、显式 sprint/watch/campus 模式、把 crawler/source/workflow 暴露给普通用户。

## 2. 身份 × 强度（取代"模式"）

### 2.1 身份 Stage（事实，决定资格 + 哪类信号当主角）

```ts
type ExperienceStage = "" | "实习" | "校招" | "社招";   // 已存在于 candidate_profiles.experience_stage（迁移 006）
```

- 来源：简历 `experience_stage`；没有则 onboarding **问一句**"你在找实习/校招/社招？"。
- 分类口径复用现有招聘类型判定（`eligibility.ts` 的 `recruitmentCategory(job)` + `hasExplicitRecruitmentType(job)`，与 `lib/insight-derive.ts` 同口径），不另造。
- 作用：
  - **资格硬门（严格同阶段，沿用 `eligibility.ts` 的 `stageState`，逐字对齐现有代码）**：用户阶段 == 岗位阶段才放行；岗位**无明确招聘类型 → `unknown` 放行**（不一刀切）；用户**没设阶段 → 不约束**。兼容矩阵：实习→实习、校招→校招、社招→社招；**跨阶段（如校招也看实习/暑期转正）V1 不做**，列为已知限制，后期可加 opt-in；
  - **信号侧重**：校招/实习 → 截止 + 校招通道开放为主；社招 → 保鲜（仍在招）+ 动量为主。

### 2.2 强度 Intensity（自调，决定推多频/多严/多少）

```ts
type RadarIntensity = "active" | "passive";   // 新增
```

- **冷启动**：onboarding 问第二句"你现在积极找、还是先看看？"，默认 `active`（蜜月期）。存为用户初始强度。
- **自调**：之后按行为校准——近期常开 + 有动作（saved/applied/打开官网）→ `active`；长期不来但有关注公司 → 衰减 `passive`；回来一动作 → 弹回 `active`。
- **手动覆盖**：偏好页一个显眼开关，用户随时改；改过则尊重手动值一段时间再恢复自调。
- 作用：
  | | active | passive |
  |---|---|---|
  | 每日量 `daily_limit` | 默认偏多（沿用 5–30 clamp） | 默认偏少 |
  | 推送频率 | 可日推 | 周报/只大事 |
  | 进清单的门槛 | 较低（含拓展） | 较高（只高价值 + 关注公司） |
  | 空状态语气 | "今天没有新的高可信机会" | "暂时没有值得打扰你的变化" |

### 2.3 蜜月期（新用户头几天）

- 新用户默认 `active`，**首屏给一次"现状盘点"**：`现在有 N 个官方确认在招、对口你的岗位`，**明确标注是"现状"不是"新变化"**，证明雷达不是空壳。
- 之后切成每日只给"变化/新进清单"；用户没动静则强度自然衰减。

## 3. 关键提醒 vs 日常推荐（强度只调后者）

| 类别 | 触发逻辑 | 例子 |
|---|---|---|
| **关键提醒（永远响，不受强度压制）** | 与用户已表达的关注强相关 | 你保存的岗快截止了 / 你保存的岗关闭了 / 你关注的公司校招通道开了 |
| **日常推荐（随强度调多少/多频）** | 系统主动筛 | 对口新进清单、仍在招的高匹配、拓展看看 |

实现：`lib/opportunities/grouping.ts` 区分两类 section；强度只影响"日常推荐"的量与门槛，不裁剪"关键提醒"。

## 4. 机会清单：信号当标签（降级，不主打）

每张卡至少有一个标签来自下表；**清单当前主要由"对口 + 刚核验仍在招"填充**，信号是锦上添花的标签，不是入选的唯一理由。

| 信号标签 | 用户文案 | 触发 | V1 状态 |
|---|---|---|---|
| **仍在招（保鲜）** | `最近确认仍在招` | active + `enrich_checked_at` 在 today SLA（24h）内（见 01 spec §2） | ✅ 主力（护城河外显） |
| **快截止** | `7天内截止` / `本周截止` / `请以官网截止时间为准` | `deadline` 可解析为日期且在窗口内（社招/实习 7 天、校招 14 天） | ✅ 上，但稀疏（约 1/5 岗有，校招约 1/4） |
| **收藏岗关闭/陈旧** | `可能已关闭` / `长时间未确认` / `建议以官网状态为准` | 用户 saved/关注对象 status∈expired/removed/error 或 stale | ✅ 关键提醒，进 saved/状态区 |
| **校招通道开放** | `这家公司校招/实习通道刚开` | 关注公司近期新增校招/实习岗（带样本门槛） | ⚠️ 校招季再做强；先简单 |
| **我们新发现** | `我们新发现的官网机会`（**禁止**"刚发布""全网首发"） | 仅当 `posted_at`（官方发布时间）可用且在窗口内；只有 `first_seen_at` 时**当前暂不可用**（6/15 重建污染，见 02 spec §4.2） | ⚠️ 暂不主打，等时间记真 |
| **公司招聘动量** | 仅 B 端/admin/高校看板用 | 见 02 spec §4.3 防假动量守则 | ⚠️ 不上 C 端"猛招" |

全局规则：
- `enrich_checked_at` 缺失（从未核验）的岗 **不进主清单**、不写"仍在招"；
- `score` 只排序，不向用户展示数字；
- 信号不足时**宁缺毋滥**，不用无关岗填满 `daily_limit`；
- **永不**出现"全网最快/最早""BOSS 没有""自动投递""没有死岗（绝对）"。

## 5. 页面与信息架构

主导航顺序：① 今日机会 `/today` ② 搜索岗位 `/jobs` ③ 关注与偏好 `/preferences` ④ 值得投 `/saved` ⑤ 已投递 `/applied`。

- `/today` = 登录后默认首页，核心产品；
- `/jobs` = 探索工具，**普通用户默认隐藏"刷新公司/发掘新公司"**（管理员或 feature flag 保留）；
- `/path`、`/me`、`/sources`、`/admin/*` 收入二级或管理员区；
- 普通用户**不出现** crawler/source/workflow/jobs_created/discovery run 等技术概念。

### 5.1 Landing `/`

首屏（benefit-led，沿用方向 v3 §0）：

```text
职达替你盯企业官网，把刚核验还在招、适合你的岗位挑出来——少踩死岗，少筛少烦。
```

三张价值卡：① `刚核验仍在招`（护城河）② `适合你`（按身份/偏好筛）③ `快截止`（校招/实习别错过窗口）。
证据文案：`每个岗位都给官方链接 + 最近核验时间 + 推荐原因。`
**禁止**：crawler 数量、全网最全、自动投递、岗位内容变更、"没有死岗"绝对说法。

### 5.2 Onboarding

1. 你在找：实习 / 校招 / 社招（简历有则带出）；
2. 你现在：积极找 / 先看看（默认积极）；
3. 目标岗位/关键词/关注公司；
4. 城市/城市偏好；
5. 学历/阶段补全；
6. （可选）传简历补技能；
7. 进今日机会（蜜月期现状盘点）。

未达 `profile_ready` 不展示随机岗位。`profile_ready` 只看身份相关字段，**强度永不卡门**。

### 5.3 今日机会 `/today`

结构：Header（身份 + 强度 + 最近生成时间）→ 信号摘要条 → 分区（按身份/强度/已触发信号重排，**不是固定三套模式布局**）→ 机会卡。

分区池（按身份/强度动态选取与排序，例）：
- `关键提醒`（快截止 / 收藏岗关闭 / 校招通道开放）——永远置顶；
- `刚核验仍在招的对口机会`（主力）；
- `拓展看看`（仅 active 强度）；
- `本周招聘动量`（仅在数据可信且非 C 端"猛招"表述时）；
- `等待再次确认`。

### 5.4 机会卡

必含：公司、岗位、城市、阶段、官方链接、信号标签、**最近核验时间**（或明确降级状态"待确认"）、2–4 条推荐原因、CTA、负反馈入口。

推荐原因示例：`方向匹配：产品经理`、`目标城市：上海`、`你关注的公司：字节跳动`、`校招岗位`、`官方源最近确认仍在招`。
**不展示**：score 数字、权重、source id、crawler method、content hash、workflow run id。

CTA：`打开官网投递`，点击行为见 01 spec §4（点击前临门核验，不阻塞）。

### 5.5 关注与偏好 `/preferences`

含：身份、强度开关、目标岗位/关键词/排除词/城市/关注公司/行业/学历、每日上限、通知设置、**关注公司覆盖状态**（covered/queued/researching/unsupported，文案见 Codex 版同口径）。
**不展示**：启动爬虫按钮、source adapter、discovery run、jobs_created。

### 5.6 值得投 `/saved`、已投递 `/applied`

- `/saved`：已保存机会工作台，显示状态变化（`最近确认仍在招`/`可能已关闭`/`长时间未确认`/`截止临近`），引导标已投/移除；
- `/applied`：投递记录，不自动判成功，岗位下线仍留 snapshot。

## 6. 用户动作与反馈

- 主动作互斥：`saved`（值得投）/ `ignored`（不合适）/ `applied`（已投递）——已在迁移 162 的 `job_actions` 增列 + RPC 实现，沿用。
- 辅助：`viewed`、`opened_official`（走 events，见 01 spec §5）。
- **ignored 必须带 `reason_code`**（白名单**精确取值**，单一事实来源 = `lib/opportunities/feedback.ts` 的 `IGNORE_REASONS` + 迁移 162 的 `job_actions_reason_code_check`，三处同改）：`role_mismatch`、`location_mismatch`、`industry_mismatch`、`seniority_mismatch`、`education_mismatch`、`compensation_mismatch`、`company_not_interested`、`already_seen_elsewhere`、`not_job_seeking`、`other`；`reason_text` ≤200 字可选。
- 反馈原则：单次不自动改偏好；连续同类反馈在偏好页提示是否调整；负反馈用于指标与降噪。

## 7. 通知与摘要

- 邮件默认关闭，用户主动开；
- 频率跟**强度**：active 默认 daily，passive 默认 weekly；用户可改；
- 摘要只含：对口新进清单、快截止、关注公司动态、saved 岗关闭/陈旧；**不发**：普通拓展、未核验的"仍在招"、内容变更；
- 防打扰：同岗 7 天不重复（除非从普通升级为"快截止"）；ignored/applied 后不再提醒同岗；passive 强度下低于高门槛不推。

---

## 8. 指标

- **北极星**：每周每活跃用户真正处理的对口机会数（打开官网/saved/applied/给原因的 ignored）。
- **信任护栏**：点击有效率（用户点开官网后岗位仍在招比例，目标 ≥99%）——埋点见 01 spec §5。
- **辅助**：feed 打开率、清单处理率、ignored 原因分布、关注公司覆盖率、核验覆盖率（24h/72h/never）。
- **不看**：总岗位数、抓取次数、jobs_created、source 数、PV。

---

## 9. 空状态与降级

- 未登录：跳 `/login?next=/today`；
- 未完成画像（身份相关字段不足）：引导设置，不展示随机岗；
- 没有机会：按强度给文案（active："今天没有新的高可信机会，可放宽城市或补关注公司"；passive："暂时没有值得打扰你的变化"），**不兜底塞最新岗**；
- source/核验元信息查询整体失败：不写"仍在招"，可降级展示普通搜索结果 + 提示"部分来源状态暂时无法确认"，记服务端 warning；
- feed 引擎异常：API 返回 503，**不返回 mock**（见技术/验收 spec）。

## 10. 产品验收要点（详见验收 spec）

1. 身份从简历/onboarding 来，决定资格与信号侧重；
2. 强度可手动设、能随行为自调、且**只调日常推荐不压关键提醒**；
3. 每张 today 卡至少一个标签 + 官方链接 + 最近核验时间或"待确认"；
4. `enrich_checked_at` 缺失/超 today SLA 的岗不进主清单、不写"仍在招"；
5. "我们新发现"在仅有 `first_seen_at`（无官方 posted_at）时不出现"刚发布"类文案；
6. ignored 必须收 reason_code；saved/applied/ignored 不重复进主清单；
7. 普通用户主动爬取入口已降级；
8. 无画像不展示随机岗、feed 失败不假成功、不展示"没有死岗(绝对)/全网最快/BOSS 没有"。
