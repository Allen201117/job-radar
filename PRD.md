# 求职雷达 PRD — 功能需求（精确版）

> 本文件是项目当前**权威的功能需求说明（PRD）**。日期：2026-06-01。
> 早期阶段路线/背景见 `求职雷达产品方案.md`；技术实现设计见 `docs/superpowers/specs/`（编写中）。

## 1. 产品定位与核心目标

面向中国求职场景的**官方岗位雷达**。核心目标：帮助用户快速发现**最新、真实、匹配**的官方岗位，减少手动刷公司官网的成本。

覆盖范围必须广，**不应只局限于少数固定公司**：
- 中国公司
- 在华外企
- 外企中国岗位（全球官网中的 China 岗位）
- 科研 / 实验室机构

## 2. 检索闭环（用户输入关键词 / 城市 / 岗位类型）

分层、按需升级：
1. **本地 jobs 岗位库检索** — 先查已有岗位库。
2. **已确认官方源刷新** — 结果不足时，刷新已验证官方招聘源（百度、京东、Apple、Siemens 等）。
3. **低频动态官方源发现** — 仍不足时，低频调用百度千帆等 search provider，发现**新的官方招聘入口**：公司官网招聘页 / 校招官网 / 社招官网 / 外企全球官网的 China 岗位页 / 官方 ATS board / 具体岗位详情页。

## 3. 来源过滤与沉淀

**必须过滤**第三方招聘平台与低质量来源：BOSS、猎聘、智联、前程无忧、LinkedIn、Indeed、知乎、公众号转载、SEO 聚合页、高校就业转载页。

- 被识别为**高质量官方招聘入口**的结果 → 沉淀到 `source_candidates`，供审核 / 解析 / 转化。
- 有 parser / adapter 支持的源 → 解析为真实 `jobs`。

## 4. jobs 写入质量门（严格）

写入 `jobs` 必须满足：
- `company`、`title`、`jd_url` 非空；
- `jd_url` 是**用户可点击的官方岗位详情页**，不能是首页 / 列表页 / 搜索页 / API JSON；
- 页面 HTTP 200，且包含岗位 title 或核心标题片段；
- 同一 `jd_url` 不重复插入。

## 5. 覆盖扩展（持续扩大公司与 source）

不应停留在百度、京东、SHLAB 等少数源，应持续通过 `source_candidates` + known sources refresh + parser conversion 扩大可用岗位来源。

- **中国公司**：互联网大厂、硬科技、新能源、制造业、银行券商、央国企 —— 字节、腾讯、阿里、美团、华为、比亚迪、宁德时代等。
- **外企（在华 / 全球官网 China 岗位）**：Apple、Microsoft、Amazon、Siemens、Bosch、Tesla、J&J、AbbVie、Merck、Roche、Novartis 等。
- **招聘系统（ATS）**：Moka、北森、飞书招聘、Workday、SuccessFactors 等。

## 6. 前端岗位看板

- 展示：公司、岗位标题、城市、岗位类型、匹配分、岗位详情链接。
- 用户操作：`saved` / `ignored` / `applied`，**按用户隔离**。

## 7. 简历画像驱动匹配

- 用户粘贴简历文本，或上传 `.txt` / `.md`。
- 规则解析：目标岗位、目标城市、技能、行业、求职阶段、教育 / 经历摘要。
- 用户确认后同步到 `user_preferences`，用于岗位匹配评分与排序。
- **边界**：简历功能只服务岗位匹配，不扩展为简历优化器，不接 LLM，不做自动投递。

## 8. 受控实时检索策略（百度千帆等）

定位 = **官方招聘源发现工具**，不是逐岗位搜索器。
- 默认一次用户点击只调用 **1 个** generated query；点击「继续发现更多」才调用第 2 个；
- 相同 query 在缓存时间内复用结果；
- 遇 429 进入 cooldown；
- 每次调用记录 `discovery_runs`，并沉淀 `source_candidates` / `jobs`；
- 无结果必须给**可解释 `failure_reason`**：`provider_rate_limited`、`all_results_rejected`、`candidates_pending`、`parser_missing`、`quality_gate_failed` 等。

## 9. 最终目标（闭环）

形成 **本地岗位库检索 + 已确认官方源刷新 + 低频动态官方源发现 + 简历画像匹配排序** 的闭环，让用户看到**真实、最新、匹配、可点击投递**的官方岗位。
